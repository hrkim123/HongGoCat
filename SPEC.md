# HongGoCat — 사양 · 요청 이력 · 결정 사항

> **작업 규칙**: 수정 요청이 오면 **먼저 이 문서를 읽고**, 여기 확정된 동작(§2)을 깨지 않도록 작업한다.
> 새 요청/수정은 아무리 사소해도 §2·§3에 추가·갱신한다. (회귀 방지)

최신 갱신: 2026-07-13

---

## 1. 개요
- **HongGoCat**: Steam Bongo Cat 스타일의 **전체화면 데스크톱 오버레이**. 키보드/마우스 입력에 맞춰 고양이가 연주. 방 코드로 친구와 멀티, 커서 유도 미사일로 서로 맞히기.
- 스택: Electron(메인) + Canvas 렌더 + `uiohook-napi`(전역 입력) + `ws`(릴레이 서버).
- 실행: `npm install` → 서버 `npm run server` → 오버레이 `npm start`.

## 2. 확정 동작 (⚠️ 회귀 금지 — 요청 없이 되돌리지 말 것)

### 창 / 오버레이
- **전체화면 투명 오버레이**. 한 번에 **한 모니터**를 덮음(Windows가 투명창을 단일 모니터로 클램프하므로 다중 모니터 동시 span 불가).
- **작업 표시줄까지 덮어야 함**: `resizable:false`면 Windows가 창을 **작업영역(work area)으로 클램프**해 작업 표시줄(하단 48px)을 못 덮음 → 개미가 작업 표시줄 위에 뜨고 미사일이 작업 표시줄에 반응 못함. **해결**: `resizable:true`(frame:false라 사용자용 리사이즈 그립은 없음) + `ready-to-show`에서 `win.setBounds(activeDisplay().bounds)`로 **전체 모니터(bounds) 크기 재적용**. 실측: 창 2560×1440(작업표시줄 포함) 확인. ⚠️ `resizable`를 false로 되돌리면 작업표시줄 미커버로 회귀.
- **클릭 통과(click-through)**가 기본. 커서가 **고양이 위젯(hotzone) 위에 있을 때만** 클릭 가능(메인 프로세스가 실제 커서를 24ms로 폴링해 `setIgnoreMouseEvents` 토글).
- **상단 흰 바 — 진짜 원인·해결 확정(픽셀 측정으로 규명)**:
  - 원인: **투명 창의 GPU 합성**이 창이 활성(foreground)일 때 y=0에 ~22px 흰 바를 그림(Electron+Windows 투명창 버그). 픽셀 측정으로 활성 시 (250,253,255) 바 확인.
  - **해결(핵심)**: `main.js` 맨 위 **`app.disableHardwareAcceleration()`** → 활성/클릭/설정열기 등 어떤 상태에서도 바가 안 뜸(강제 활성화 테스트로 확인). ⚠️ **이 줄을 지우면 흰 바 재발** — 절대 제거 금지.
  - 시도했으나 효과 없던 것들(다시 하지 말 것): `focusable:false`(오히려 항상 바), WS_EX_NOACTIVATE(직접), 1px 축소(최대화 회피), DWM BORDER_COLOR/NCRENDERING, blur-on-focus/poll-blur.
  - 보조로 남긴 것: DWM `BORDER_COLOR=NONE`+`CORNER_PREFERENCE=DONOTROUND`(테두리/둥근모서리), 시작 시 `showInactive`, 커서 hotzone로 클릭 통과 토글.
  - 참고: HW 가속 off → 캔버스 소프트웨어 렌더. 가벼운 오버레이라 OK지만, 렉 발생 시 최적화(유휴 시 FPS 다운 등) 여지.
- **단일 인스턴스**만 실행(중복 실행 시 기존 창 포커스). 유령 창 방지.
- 창 타이틀 비움.

### 캐릭터 (고양이 전용)
- **고양이만** 지원(개/돼지/원숭이 제거됨 — 되살리지 말 것).
- 스타일: 흰 기본, 분홍 볼터치, 감은 눈웃음 + 자연스러운 눈 깜빡임(랜덤 윙크 아님), 외곽선, 꼬리(살랑), 광택 하이라이트, 그라데이션 입체감.
- **팔(몸통→발 연결선) 없음**. 앞발은 작은 패드만. 키 입력 시 좌/우 앞발이 키보드를, 마우스 클릭 시 오른발이 마우스로 이동해 누름.
- 위젯 = [고양이 + 책상(키보드/마우스) + 하단 바]. 하단 바 폭 = 고양이 폭(정렬 일치). `SCALE=0.8`(캐릭터·책상·바 20% 축소, **카운트 텍스트는 고정 크기 유지**).
- **위치 = 드래그-스냅 프리셋**: 드래그로 옮기되 놓으면 **가장 가까운 프리셋 앵커로 스냅**. 앵커 = `GRID_COLS×GRID_ROWS`(현재 6×4) 격자, 사용 가능한 화면 범위(0..maxX,0..maxY)를 균등 분할한 **위젯 top-left** 위치. 드래그 중엔 앵커 점을 캔버스에 흐리게 표시(가장 가까운 건 파란 하이라이트). 비율 기반이라 해상도 달라도 일관.
- **작업 표시줄 회피**: 세로 범위의 바닥 = `usableBottom()`(=작업 표시줄 top, 없으면 화면 바닥). 그래서 프리셋·드래그·상대 캐릭터 모두 **작업 표시줄과 겹치지 않음**(캐릭터는 표시줄 위 경계까지만). 개미는 표시줄 위에서 그대로 논다.
- **프리셋 유지**: 고른 앵커를 그리드 셀 `{c,r}`로 `localStorage['anchor']`에 저장. **모니터 이동·앱 재시작 후에도** 같은 프리셋 셀을 새 화면에 다시 적용(`onLayout`→`wx=null`→`initWidget`이 저장된 셀 재적용). 모니터 이동은 미사일/개미 등 멀티 상호작용에 영향 없음(전부 정규화 좌표, ws 연결 유지).
- **이름 = 명패(nameplate)**: 책상 앞쪽 하단에 **어두운 둥근 명패 + 밝은 글씨**(`#ffe9c7`)로 그려 가독성↑, 키보드와 분리. 키보드는 살짝 확대(`kbW 92→104, kbH 22→24`).

### 하단 바 / 카운터 / 메뉴
- 하단에 **둥근 UI 바**: 왼쪽 **카운터**(햄버거 옆까지 넓게), 오른쪽 **햄버거(≡)**.
- **카운터** = 내 상호작용 누적 횟수. **애니메이션 없음**(통통 튀지 않음). localStorage 저장·누적.
- 카운트 규칙: **키는 처음 누를 때 1회만**(꾹 눌러도 반복 안 됨), **클릭 1회**, **스크롤(휠)은 카운트 안 함**.
- **하단 바 폭 = 책상(desk) 폭**: 둘 다 `cellPxW`에서 파생(HUD `SIDE=0`), SCALE 바뀌어도 **항상 동일 폭·정렬**. (책상이 바보다 크면 안 됨 — 실측 diff 0 확인.)
- **레이어 분리(무기 > 캐릭터 UI)**: `#hud-bar`가 DOM이라 캔버스 위에 떠서, 개미/미사일이 캐릭터 밑(바 위치)으로 오면 가려졌음. → **무기 전용 상위 캔버스 `#fx`**(DOM상 `#hud-bar` 뒤 = 위, `pointer-events:none`) 추가. 렌더 루프에서 `ctx`를 두 레이어로 스왑: **stage(=고양이·프리셋점·상대카운터)** 먼저, **fx(=쉴드·미사일·이펙트·작업표시줄FX·개미)** 를 그 위에. → 무기가 캐릭터 UI보다 항상 위에 보임. HUD 클릭은 fx가 pointer-events:none이라 그대로 됨.
- **햄버거 → 설정 창 토글**(열려 있으면 닫힘). 설정 창은 오버레이와 **같은 최상위 밴드(screen-saver)** 에 두고 포커스 시 `moveTop` → 투명 오버레이 뒤로 내려가 "사라져 보이던" 문제 방지(예전엔 floating이라 오버레이(screen-saver) 아래로 가끔 가려짐).

### 채팅
- `Ctrl+Shift+B`(또는 설정의 채팅 버튼) → 입력창. Enter 전송/Esc 취소. 머리 위 말풍선(최대 80자). 채팅 때만 잠깐 창을 포커스 가능하게 함.
- 말풍선: 글씨 16px(위젯 축소 감안 확대), **머리 바로 위에 앵커**(`y=headTopY-h-14`, 예전엔 셀 상단이라 너무 높았음), 텍스트 **세로 중앙 정렬**.
- 이름 명패: 17px로 확대(예전 12px는 축소 후 너무 작음), 키보드 높이 살짝 줄여(kbH 24→20) 자리 확보.

### 무기 슬롯 / 미사일
- **무기 슬롯 3개**: `Ctrl+Alt+1 / Ctrl+Alt+2 / Ctrl+Alt+3`. 각 슬롯에 무기를 설정 창에서 지정. 단축키 누르면 그 슬롯 무기 발사. uiohook로 **초기 press 1회만**(꾹 눌러도 1발). ⚠️ 전역 훅은 키를 **감지만 하고 막지 못함** → 조합이 OS 단축키와 겹치면 그 동작도 같이 실행됨. **Ctrl+숫자=브라우저 탭, Ctrl+Shift+숫자=탐색기/바탕화면 아이콘 크기 변경**과 겹쳐서 → **Ctrl+Alt+숫자**(표준 단축키 아님)로 확정. uiohook 키코드는 물리키 기준이라 조합 기호와 무관하게 숫자키로 매칭.
- 무기 dispatch는 `fireWeapon(id)`에서 분기(확장 지점). `me.slots`는 localStorage 저장.
- **무기 종류**: `none` / `missile`(🚀) / `shield`(🛡) / `ant`(🐜). `WEAPONS` 등록 + `fireWeapon(id)` 분기로 확장.
- **🛡 쉴드**: 캐릭터 중앙 부채꼴이 **아니라**, 캐릭터에서 **떨어져 떠 있는 실제 방패 플레이트**. 커서 방향으로 `SHIELD_DIST=118×scale` 만큼 바깥에 떠서 캐릭터↔위협 사이를 막음(곡면 판 + 외곽 rim + 가운데 보스). 각폭 `SHIELD_SPAN=86°`, 두께 `SHIELD_T`. **10초 지속 → 3초 쿨타임**. 미사일이 플레이트 반경 근처(`SHIELD_DIST±SHIELD_BAND`)에서 **안쪽으로 들어올 때** 막힘(바깥으로 발사는 통과 → 방패 뒤에서 공격 가능).
  - **체력(HP=10)**: HP가 줄수록 **방패 판에 균열이 번지고**(`nCracks=round((1-hp)*8)`) 색 **청록→빨강** + HP<35% 깜빡임. **숫자 UI 없음**. HP 0 → **유리 파편 파괴 연출** 후 소멸(쿨타임 파괴시점부터).
  - **데미지 = 미사일 power**: 합쳐진 미사일(power N)은 쉴드에 **N만큼** 데미지(`hitMyShield(power)` / `shield-hit`에 power 전송). power-5 미사일 1발 = HP 5 감소.
  - 종료 시엔 마지막 1초간 **점점 투명 + 깜빡임**으로 자연스럽게 사라짐(파괴가 아닌 시간만료).
  - **막기 규칙(중요)**: `shieldBlocks(p, now)`는 미사일이 아크·반경 안에 있고 **캐릭터 쪽으로 들어오는(inward, 속도·방향 내적>0)** 경우에만 막음. → 바깥으로 **발사되는 내 미사일은 통과**(공격 가능), **들어오는 미사일(내 것이 커서를 향해 되돌아오거나 상대 미사일)은 막힘**. 자기 쉴드도 막으므로 **솔로에서도** 커서를 자기 쪽으로 끌면 방어막 파손을 확인 가능. 내 미사일이 상대 쉴드에 막히면 `shield-hit`을 그 상대에게 전송, 내 쉴드가 막으면 로컬에서 HP 차감.
  - 멀티: 쉴드 상태(각도+남은시간+**hp/max**)를 90ms로 브로드캐스트 → 상대 화면에도 균열까지 렌더. 파괴 시 `{ttl:0, broke:true}`로 파편 연출 동기화. 서버 릴레이: `shield`(hp/max/broke 포함), `shield-hit`.
- **미사일 활성 개수 = 파워 합산**(합쳐진 power-5 미사일 = 5로 카운트). 상한 `MAX_MISSILES=6`(총 파워). 터지거나 사라져 파워가 줄면 재발사 가능.
- **미사일**(기본 슬롯1=미사일, `Ctrl+Alt+1`). 아래 동작 유지:
- **자기 고양이 머리 위**에서 발사 → **마우스 커서를 유도 추적** → 고양이에 충돌 시 폭발 + **X눈·별 리액션**.
- **좌클릭 부스터**: 마우스 **왼쪽 클릭** 시 활성 미사일이 **현재(마지막) 진행 방향으로 고정**되어 **3배 속도로 직진**(`BOOST_MULT=3`, 유도 해제) + **부스터 화염 연출**(긴 파란 이중 화염 + 스파크). main의 uiohook mousedown `button===1` → `command{t:'boost'}` → `boostMissiles()`. `p.boost` 상태로 `stepProjectiles`에서 유도 스킵. **오버레이 밖으로 나간 미사일은 즉시 제거**(경계 밖 60px)해 활성 개수에서 빠지므로 바로 재발사 가능.
- **동시 최대 6발**, 유지시간 **14초**. 터지거나 사라지면 다시 발사 가능.
- **병합**: 미사일끼리 가까워지면 합쳐져 `power`↑ → **크기 커지고 빨강→황금색 + 반짝임 + 더 큰/황금 폭발**.
- **멀티**: 내 미사일 위치를 **내 고양이 기준 상대좌표**로 90ms마다 브로드캐스트 → 상대에게도 보임. 내 미사일 ↔ 상대 미사일 충돌 시 폭발. 내 미사일이 상대 고양이 명중 시 `hit` 전송 → 상대 화면에서도 반응.

### 🐜 개미 무기 + 작업 표시줄 파괴 연출 ("바탕화면 부시기" 참고)
- **작업 표시줄 "파임(dig)" FX**: 미사일 폭발이 작업 표시줄 영역에서 일어나면 **그 자리가 점점 깊게 파임**(슈팅게임 땅 파이듯). `carve[]` 높이맵(열당 `CARVE_SEG=6px` 깊이)에 `carveTaskbar(x,power)`로 크레이터 누적(반복 타격 시 심화, 최대 ≈표시줄 높이). `drawTaskbarDig`가 표면(tb.top)↔파인 프로필 사이를 어두운 구덩이+거친 테두리로 그림 + 파편(debris). **영구 저장**(`localStorage.bardig`, 1.5s 스로틀). **복구**: 설정창 “🧱 작업 표시줄 복구” → `reset-taskbar`→`resetTaskbarDig`. 실측: 반복 타격 크레이터 34px, 약한 타격 14px, 미타격 0(온전).
- **개미(🐜)**: 커서 위치에 소환 → 작업 표시줄로 낙하 → **작업 표시줄 경계선 위(top)에 발을 딛고**(`antGroundY = taskbarRect().top - 5×scale`, 안으로 파묻히지 않음) 배회, 상대 개미가 있으면 근접해서 서로 공격.
  - **작업 표시줄 자동 대응**: `taskbarRect()`가 OS 작업영역(workArea) vs 모니터 bounds 차이로 계산 → 사용자마다 다른 **하단 작업 표시줄 높이/유무에 자동 적응**. (자동숨김이면 workArea==bounds라 미검출 → 화면 최하단에 안착. 상/좌/우 작업 표시줄은 미지원 — 하단 기준.)
  - **인원당 최대 5마리**(`MAX_ANTS`), **체력 3**(`ANT_HP`). 근접 공격 1대(600ms 쿨), 3번 맞으면 **사망(피 튀김 debris + 잠깐 시체 후 소멸)**. 피격 시 머리 위 **HP 점(pips)** 표시.
  - **미사일로도 처치 + 범위(AoE) 공격**: 미사일이 개미/고양이/상대 미사일/**작업 표시줄**에 닿으면 폭발하며, **폭발 시 `explode(x,y,power)`가 폭발 이펙트 반경(`blastRadius` — power에 비례) 안의 모든 대상에 데미지**. 즉 한 발로 반경 내 여러 개미를 동시 처치. 고양이는 쉴드가 폭발 지점을 덮고 있으면 보호(`catShieldCovers`). 미사일이 작업 표시줄에 닿으면 폭발→균열.
  - 각 클라이언트가 **자기 개미**를 시뮬레이트(낙하·AI·HP). 내 개미는 **항상 가장 가까운 상대 개미로 진격**(거리 제한 없음), 근접(≤22px) 시 `ant-hit` 전송 → 상대가 자기 개미 HP 차감(소유자 권위). **개미 색 = 소유자 털색**(`ANT_COLORS[skin]`)이라 플레이어마다 다르게 보임.

### 🕳 블랙홀 무기 + 업적/해금 + 호스트
- **블랙홀**(`blackhole`): 커서 위치에 **10초 고정** 소환, **쿨타임 60초**. 반경 `BH_R=0.12×화면폭`(≈프리셋 한 칸). 반경 안의 **미사일·개미를 중심으로 끌어당김**(`blackholePull`: 힘 ∝ 중심에 가까울수록↑), 코어(`BH_CORE`)에 닿으면 **입자화(`bhDust`)돼 소멸**. 캐릭터는 안 빨림.
  - **미사일 네트워크 처리(핵심)**: 블랙홀을 정규화 좌표로 broadcast(`{t:'blackhole',nx,ny,ttl}`). **각 클라이언트가 자기 미사일/개미에만** 모든 활성 블랙홀(내 것+상대 것) 기준 힘을 적용 → 소유자만 움직여 충돌 없음, 해상도 무관. 소멸도 소유자가 처리.
  - 시각: 보플배틀풍 소용돌이(검은 코어 + 보라 강착원반/나선/글로우), fx 레이어. broadcast로 모두 렌더.
- **업적 "🐜 개미 학살자"**: 내 미사일/블랙홀로 **개미 100마리 처치**(`antKills` localStorage 누적, 자기 개미 포함 → 솔로 테스트 가능) → **블랙홀 해금**. 달성 시 토스트. 설정창 **🏆 업적 팝업**(진행도 바 + 상태). 무기 슬롯에서 블랙홀은 미해금 시 **🔒 잠금(비활성)**.
- **호스트 = 전 무기 해금(서버 판별)**: 서버가 **loopback(=서버와 같은 PC, `ws://localhost`)** 접속 클라를 `host:true`로 `joined`에 전달 → 그 PC는 `localStorage.host=1` 저장, 업적 무관 전 무기 사용. 친구는 다른 IP라 위장 불가. 오프라인/비호스트는 업적 게이팅.
- **디버그(호스트 전용)**: 업적 팝업에 `＋10 처치`/`업적 초기화` → `achv-add`/`achv-reset` IPC.

### 멀티플레이 (서버 릴레이)
- `server/server.js`: 방 코드 입장(방당 최대 12명, `MAX_ROOM_SIZE`). 릴레이 메시지: `join/roster/pos/pulse/chat/throw/missiles/hit/shield/shield-hit/ants/ant-hit/blackhole/update`. `joined`에 `host`(loopback 여부) 포함.
- **접속자는 각자 자기 위치에 표시(일렬 배치 아님)**: 각 클라이언트가 자기 위젯 위치를 **화면 정규화 좌표(nx,ny=wx/W, wy/H)** + `taps`로 90ms마다 `pos` 브로드캐스트. 상대는 그 값을 자기 화면 크기에 맞춰(`nx*W, ny*H`) 렌더 → 상대가 놓은 자리에 뜨고, 각자 드래그로 독립 이동. **상대 카운터는 상대 캐릭터 아래에 캔버스로 표시**(`drawPeerCount`; 내 것은 DOM HUD). 디자인은 **내 하단바와 동일**(밝은 바 + 어두운 카운터 칩, 전체 셀 폭), **햄버거만 없음**. ⚠️ 이 기능은 서버도 최신 코드(`pos` 릴레이)여야 동작 — 서버 재시작 필요.
- **원격 엔티티 보간(끊김 방지)**: 상대 미사일·개미는 ~90ms마다 오는데, 수신 측에서 매 프레임 최신 위치로 **부드럽게 lerp(`SMOOTH=0.4`, 브로드캐스트 50ms)** 하며 따라가 순간이동처럼 안 보이게 함. `remoteMissiles/remoteAnts`는 id별 지속 Map(`items`)에 표시좌표(sx,sy)를 유지(`mergeRemote`), 미사일은 이동 방향으로 회전(스핀 대신).
- **미사일·개미 좌표 = 정규화 화면 좌표(핵심)**: 캐릭터 기준 픽셀 오프셋이 아니라 **미사일=(nx,ny)=(x/W,y/H)**, **개미=nx=(x/W)** 로 전송. 수신 측은 미사일을 `(nx*W, ny*H)`에, 개미는 `(nx*W, 내 작업표시줄선)`에 렌더. → 쏜 사람이 조준한 지점의 비율 = 대상 캐릭터의 앵커 비율이므로, **해상도/화면비가 달라도 미사일이 대상에게 정확히 수렴**하고 개미는 항상 **내 작업 표시줄 위**에서 논다. (서버는 필드 이름과 무관하게 그대로 릴레이 — 서버 변경 불필요.)
- `joined` 메시지에 `max`(방 정원) 포함. 오버레이는 `roster.peers.length`로 **현재 인원**을 집계해 설정 창에 push → 설정 창 멀티플레이 섹션에 **"접속 인원: N / 12명"** 표기(오프라인이면 "— / 12명 (오프라인)"). 인원은 roster 갱신·접속·해제 시 실시간 반영.
- 프라이버시: 네트워크로는 입력 **종류(key/mouse)** 와 미사일/채팅만 전송. 어떤 키인지는 전송 안 함(초당 25펄스 제한).

    - **자동 업데이트 타이밍**: 시작 시 체크·다운로드 → 다운로드 완료가 **실행 후 3분 이내면 즉시 silent 설치+재실행(=켤 때 업데이트)**, 그보다 오래 켜져 있었으면 **종료 시 설치**(세션 중 강제 재시작 방지). CI는 `releaseType: release`로 초안 없이 바로 공개.
    - **실행 중 새 버전 알림**: `initAutoUpdate`가 **30분마다 checkForUpdates** → 실행 중 새 릴리스가 나오면 다운로드 후 (>3분 세션) **OS 알림(`Notification`, 클릭 시 즉시 재시작 적용) + 오버레이 토스트(`#update-toast`, 정보용·pointer-events:none)** 표시.
