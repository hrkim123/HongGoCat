## 0. 개발/노출 정책 (중요)

- **완성 전까지 미배포.** `npm version` / 태그(`v*`) / GitHub 릴리스 **금지** (그래야 유저에게 안 나감).

## 1. 개요 / 목표

- 냥코대전쟁 스타일 **1v1 라인 대전**. 두 플레이어가 작업표시줄(레인)의 양 끝 진영에서 시작.
- **마나 코스트**로 유닛을 소환 → 유닛은 상대 기지 방향으로 **자동 진군** → 만나면 전투.
- 기존 **무기(미사일·게틀링 등)**로 다가오는 적 유닛을 **방어**.
- **상대 기지(고양이 HP) 파괴 = 승리.**
- 유닛은 **가챠**로 획득, **덱 최대 10장**으로 편성해 입장(배틀 HUD: 앞 5 활성 + 뒤 5 벤치, 판 중 열 단위 스왑).
- **베팅:** 신청 시 **3가지 재화(🪙카운트/💎젬/🔩강화부품) 중 택1 + 금액**을 걸고, 이긴 사람이 가져감. 양쪽 다 해당 재화를 낼 수 있어야 성립(부족 시 토스트 안내).

## 2. 매치 흐름

0. **덱 조건(필수):** 배틀 참여는 **덱 완성(소환체 ≥3, 무기 ≥1)** 상태에서만. `BattleGacha.deckReady()`.
   - 신청자 덱 미완성 → 신청자에게 토스트 "덱 구성을 완료하세요 (소환체 3+, 무기 1+)".
   - 수신자 덱 미완성 → 신청자에게 토스트 "덱 구성이 안 된 상대입니다.".
1. **신청:** 신청자가 상대 피어 + **베팅 재화 종류(🪙 카운트 / 💎 젬 / 🔩 강화 부품 중 택1) + 금액** 지정 → `battle-req{ betCur, betAmt }`.
   - 신청 전 **내 잔액 확인:** 내가 해당 재화를 `betAmt`만큼 못 내면 토스트 "재화가 부족합니다" (신청 안 나감).
   - **상대 잔액 확인(수신자 측):** 수신자가 `betCur`를 `betAmt`만큼 못 가지고 있으면 자동 거절 + 신청자에게 토스트 "상대가 [재화]가 부족합니다".
2. **수락/거절:** 상대에게 팝업 표시. 수락 → `battle-acc`, 거절 → `battle-decline`(reason: deck/poor 등).
3. **진입:**
   - 두 참가자 외 **모든 피어 숨김 + 상호작용 차단** (배틀 모드 플래그 on).
   - 각 참가자 고양이가 자기 **레인 끝(기지)**으로 자동 이동.
   - 마나 0에서 시작, 베팅액 양쪽 escrow(예치).
   - **시작 카운트다운:** 화면은 즉시 배틀 뷰로 전환되지만 게임은 바로 시작 X. 화면 중앙 **3 · 2 · 1 → START!**(각 0.8s, 총 3.2s) 뒤 전투 시작. 카운트다운 동안 마나·행군·소환·무기 모두 정지.
4. **전투:** 유닛 소환/무기 사용으로 상대 기지 HP를 깎음.
5. **종료:** 한쪽 기지 HP 0 → **결과 연출**(화면 중앙 **WIN**=파란색+색종이 빵빠레 / **LOSE**=빨간색, 딤 배경) **3초** → 원래 방으로 복귀(피어 복원), 베팅 정산.
6. **정산 규칙 (확정):**
   - **정상 종료:** 양쪽이 보고한 결과가 **일치할 때만** 승자에게 팟 정산.
   - **본인 의사로 이탈**(나가기/포기 버튼, 배틀 모드 명시적 종료) → 남은 유저 **승리 판정**. **나가기 버튼은 즉시 종료 X → "정말 나가시겠습니까? (나가면 패배 처리) YES/NO" 확인 팝업**. YES = 자발적 패배(LOSE 연출 후 복귀).
   - **시스템 오류·비자발적 팅김**(예상치 못한 ws close/타임아웃, leave 메시지 없이 끊김) → **베팅 무효(환불)**.
   - 판별 휴리스틱: 명시적 `battle-leave` 수신 = 자발적 이탈(패배). 그 외 갑작스런 연결 끊김 = 비자발적(무효). 완벽 판별은 불가하므로 이 기준을 사용.

## 3. 좌표계 & 해상도 독립 (핵심 설계)

플레이어마다 화면 해상도가 달라 픽셀 거리가 다름 → **픽셀을 와이어에 절대 싣지 않는다.**

- **레인 좌표 `L ∈ [0,1]`**: `L=0` = side0 기지, `L=1` = side1 기지. **와이어에는 `L`만 전송.**
- **렌더 시 각자 화면폭에 매핑 + 관점 뒤집기:**
  - 내가 side0 → `screenX = L × laneWidthPx`
  - 내가 side1 → `screenX = (1 − L) × laneWidthPx` (내 기지가 항상 내 화면의 같은 쪽에 보이게)
- **속도/사거리 단위는 픽셀이 아니라 레인비율.** 예: `speed 0.15` = 초당 레인의 15% 이동. → 화면 넓이와 무관하게 **레인 횡단 시간이 모두 동일.**
- **"동일 거리"의 정의:** 맞춰야 하는 건 픽셀 거리가 아니라 **레인 진행도/시간**. 픽셀 거리는 화면마다 달라도 게임적으로는 완전히 같다.
- **side 결정:** 신청자=side0, 수락자=side1 (또는 netId 낮은 쪽=side0). 리틀보이식 결정적 tie-break.

### 지형(땅 파임) & 구멍 — 배틀에서도 유지
- 작업표시줄 파임(carve)·실제 구멍 그대로 유지. **참호로 적 지상 진군을 막는 전략 요소**(내 유닛도 빠지는 양날).
- **구멍 위치·크기도 `L`(정규화)로 표현·방송.** → 양쪽 화면에서 "같은 L의 유닛이 같은 구멍에 빠진다"가 동일하게 참(해상도는 픽셀 수만 바꿀 뿐 판정 불변).
- **지상 유닛:** 구멍 만나면 낙하/제거(기존 `mechaOverHole` 로직 재사용). **공중 유닛(`flying:true`):** 구멍 무시.
- **낙하 판정 = 소유자 권한:** 공유된 L 구멍 데이터로 소유자가 판정 → 결과 방송(일관성 원칙). 세로 낙하 애니메이션은 로컬 연출, "빠졌다/제거"는 모두 동일.

## 4. 마나 / 코스트

- 마나 **최대 10**, 회복 **+0.5/초** (0→풀차지 20초).
- 유닛 소환 = 해당 유닛 `cost` 만큼 마나 소모.
- 무기 시스템도 마나 코스트 사용(미사일 0.5/발 등).

## 5. 덱

- **덱 슬롯 최대 10장.** 보유 유닛 중 최대 10개 편성(배틀 HUD는 앞 5 활성 + 뒤 5 벤치 실루엣, 벤치 탭 시 같은 열 앞뒤 스왑 — 냥코풍).
- 배틀 중 앞줄 5칸을 버튼/단축키로 소환 (마나 충분 시). 뒷줄 5칸은 실루엣, 탭하면 앞줄과 교체.

## 6. 유닛 데이터 스키마 (하드코딩 금지 → 데이터 정의)

유닛 = **데이터 한 줄**, **범용 시뮬레이터 1개**가 이동·타겟·공격 처리.
속도/사거리/범위는 전부 **레인비율 단위**(3장 참고).

**실제 데이터 정본 = `renderer/battle/units.js`** (`window.BattleData`). 아래 표는 밸런스 초안(사람이 보는 요약).

**시뮬레이터 개념:** 매 프레임 각 유닛 `L += speed*dt`(기지 방향), 사거리 내 가장 가까운 적 유닛/기지 찾으면 `atk.type`별로 `cd`마다 공격. 렌더는 `art` 필드로 기존 draw 함수에 위임. → **유닛 추가 = 데이터 추가**, 코드 무증가.
지상 유닛은 매 프레임 자기 `L`이 구멍 `L`-범위에 걸리는지 검사(낙하). **`flying:true`** 유닛은 구멍 검사 스킵(3장 참고).

### 디자인 컨셉

## 7. 소환체 vs 무기 시스템 구분

- **유닛(cat:'unit'):** 덱에 편성, **가챠로 획득**. 자동 진군.
- **무기(cat:'weapon'):** 덱 불필요, 배틀 중 마나 코스트로 사용. **획득도 가챠로 전환**(아래 참고).

### 기존 무기 → 가챠 전환 (경제 변경)
**지금 존재하는 오버레이 무기들의 획득을 "카운트 구매"에서 "가챠"로 전환한다.** (카운트 구매 폐지, 대신 카운트→젬 치환)
이는 오버레이 게임 무기에도 적용되는 큰 변경 → dev 게이트 뒤에서 함께 개발, 완성 전 미배포.

**⚠️ 1회 초기화 마이그레이션:** 배틀 모드 버전 최초 실행 시, **기존 획득자 포함 모든 유저의 카운트·보유 무기·업그레이드를 전부 초기화**한다(localStorage 마이그레이션 플래그 1회). 초기화 후 남는 건 **기본지급(개미 유닛 + 미사일 무기)** 뿐, 나머지는 새 가챠 경제로 처음부터 획득.

- **기본지급:** 개미(유닛) + 미사일(무기). 미사일의 오버레이 업그레이드(개수↑·합체)는 유지 → 배틀 밸런스에 반영.

## 8. 가챠 / 컬렉션

- **가챠 재화 = 💎 젬** (임시명, 카운트와 분리). **카운트 10,000 → 젬 1개로 치환**(교환 버튼). 1회 뽑기 = 💎1.
- **진입:** 상점에 **[소환] 버튼** 추가 → 누르면 **가챠 팝업**이 뜬다. **1회 뽑기(💎1)** + **10연 소환(💎10)** 지원.
- **10연 소환:** 💎10 소모, 각 개별 연출을 순차 재생. 우상단 **SKIP ⏭**(버튼 아닌 텍스트형) 누르면 즉시 **결과 그리드**로. 결과 = **상단 5 / 하단 5 = 10개**를 한눈에(아이콘·이름·희귀도·신규/🔩중복). 좌상단에 진행 카운터(`n/10`).
- **풀:** 기본지급(개미·미사일) 제외한 **유닛 + 무기 전체**가 한 풀에 섞여 나옴(추후 배너 분리 가능).
- **희귀도 4단계 & 확률(tier):** 일반 45% / 고급 40% / 희귀 13% / 전설 2%. **tier 선택 후 그 안에서는 균등 분배(엔빵)** → 같은 희귀도에 항목이 늘면 개별 확률 = tier% ÷ 개수. (천장은 추후)
- **희귀도별 연출 구분 (내가 제작):**
  - 🩶 **일반** — 흰 섬광 + 카드 슥 등장(짧게)
  - 🔵 **고급** — 파란 빛기둥 + 반짝
  - 🟣 **희귀** — 보라 소용돌이 + 파티클 확산
  - 🌟 **전설** — 금/무지개 폭발 + 화면 흔들림 + 개미 실루엣 극적 등장
- **중복 획득 = 강화 재료(🔩 강화 부품) 등급별 차등 지급** — 일반 1 / 고급 2 / 희귀 3 / 전설 5. 이 재료로 유닛/무기 **업그레이드**. 피티(천장) 추후.
- 보유 유닛·무기 + 강화 재료 보유량 **localStorage 영속**.
- **레어도 = 파워가 아니라 다양성/상황 특화.** 강함은 마나 코스트에 묶어 PvP 페이-투-윈 방지.

> 데이터 정본: `renderer/battle/units.js`의 `RARITY`(가중치/색/연출키), `GEM`(치환·뽑기 비용), `UPGRADE`(중복 시 강화 재료).

## 9. UI / HUD & 모드 표현 방식

**배틀 모드 = 새 창이 아니라 "같은 오버레이의 모드 전환(토글)".** 투명 오버레이 창 그대로, `battleMode` 상태가 켜지며 규칙/UI만 바뀐다. 바탕화면은 계속 보임(살짝 딤).

**토글 ON 시 변화:**
- 다른 피어 전부 숨김(나+상대만). **멀티 가시성 규칙:** 배틀하는 2명에겐 나머지 유저가 안 보이고 두 배틀러가 작업표시줄 양 끝으로 이동. **비배틀 유저들에겐** 방은 평소대로이되 **배틀 중인 2명의 캐릭터만 잠시 사라짐**(배틀 종료 시 복귀).
- 내 고양이 = 레인 끝 기지에 고정(자유 WASD 로밍 OFF).
- HUD 등장: 하단 **마나 바(10) + 덱 5칸(클릭/숫자키 1~5) + 무기 버튼(구매자만)**, 양 끝 **기지 HP 바**, 상단 **베팅 팟**.
- 입력 전환: 클릭/숫자키 = 소환. 소환체는 내 기지에서 나와 자동 전진.
- 딤 약 30~40%(모드 신호 + 가독성). 바탕화면 여전히 보임.

**진입 연출:** 수락 → "배틀 시작" 배너 → 두 고양이 양 끝으로 슬라이드 → 딤 페이드인.
**종료 연출:** 기지 HP 0 → 윈/로즈 3초 → 딤 페이드아웃 → HUD 제거 → 피어 복원 → 자유 오버레이 복귀.

**⚠️ 렌더링 원칙 (중요·정정):** 배틀은 **별도 캔버스/추상 도형 금지**. **지금 오버레이의 기존 draw 함수(개미·메카·인간폼 등)를 그대로 재사용**해 작업표시줄 위에서 렌더한다. 두 참가자 고양이는 작업표시줄 **양 끝**에 배치, 소환체는 내 진영→상대 진영으로 자동 전진(연출은 오버레이와 동일, 배틀 전용 규칙만 추가).
**HUD 위치:** 하단 별도 바가 아니라 **내 캐릭터 상단**에 마나 바 + 덱(소환체·무기) 표시. **마우스로 드래그 이동 가능**(위치 저장).
**작업표시줄 복구 버튼(배틀 기본 기능):** 덱과 별개로, **마나 코스트 1**의 "작업표시줄 복구" 버튼 상시 제공(파인 땅/구멍을 메워 지상 유닛 낙하 방지·역이용). 지형 파임/구멍 낙하는 전략 요소로 유지.
**종료 버튼:** 햄버거 메뉴 최하단으로 이동(설정에서 제거).
**신규 소환체:** 개미 라이플(개미+총)·개미 수류탄(개미+수류탄)·쉴더 개미 등은 **캔버스 draw 함수로 새로 제작**하고, **오버레이에서도 커서 위치에 소환** 가능해야 함(장난용).
> sim.js(전투 로직)는 유지, mode.js(별도 캔버스 렌더)는 **오버레이 통합 렌더로 재작업** 필요.

## 9b. 오버레이 ↔ 배틀 모드 관계 (중요 원칙)

**컬렉션은 공유, 동작은 컨텍스트별로 다르다.** 배틀은 "모드"이므로 홍고캣 기본 오버레이를 항상 참고해 설계한다.
- 예) **개미** — 오버레이: 업그레이드 시 소환 개수↑, 10마리 모이면 메카로 합체(현행 유지). 배틀: 덱 유닛으로 마나 1기씩 소환·자동 전진, **메카 개미는 별도 덱 유닛**(자동 합체 없음).
- 예) **미사일** — 오버레이: 업그레이드 시 개수↑ + 합체(현행 유지). 배틀: 무기(마나 0.5/발)로 발사 수/합체 임계에 업그레이드 반영.
- 업그레이드 수치는 **컨텍스트별로 적용**(오버레이=개수/합체, 배틀=유닛 스탯). 하나의 수집품이 두 모드에서 각자 규칙으로 동작.

### 소환체 오버레이 / 배틀 동작 명세

**공통 원칙**
- **오버레이:** 커서 위치에 소환(현재 개미처럼). WASD 조종형(인간)만 수동, 그 외 **공격형은 가장 가까운 상대 캐릭터(고양이)를 자동 조준·공격**, **근접 전용은 작업표시줄 배회하다 적 소환체와 교전**.
- **배틀:** 내 기지에서 자동 전진 → 사거리 내 적 유닛/기지 공격(sim.js). 지상 유닛은 구멍 낙하, 공중은 무시.
- **렌더/연출은 두 모드 동일**(기존 오버레이 draw 재사용). 배틀 전용 규칙만 추가.

| 소환체 | 오버레이 동작 | 배틀 동작 |
|---|---|---|
| 개미 | 배회 + 근접 교전 | 근접 전진 |
| 개미 라이플(신규) | 가까운 적 캐릭터에 3연발 사격 | 원거리 3연발 전진 |
| 개미 수류탄(신규) | 가까운 적에 수류탄 광역 | 광역 전진 |
| 쉴더 개미(신규) | 방패 든 개미 배회(공격X, 몸빵) | 탱커 전진 |
| 정찰 개미(신규) | 빠르게 배회 + 근접 | 고속 러셔 |
| 폭탄 개미/카미카제(신규) | 가까운 적에 돌진→자폭 광역 | 접근 자폭 |
| 의무 개미/메딕(신규) | 근처 아군 소환체 회복 | 아군 회복(후방) |
| 말벌 드론/비행(신규) | 공중 배회 + 원거리 | 공중 전진(지상 근접 무시) |
| 얼음 개미/프리저(신규) | 적에 둔화 탄 | 둔화 컨트롤 |
| 일개미/워커(신규) | (오버레이선 배회만·장난용) | 정지형·마나 회복 |
| 지휘 개미/커맨더(신규) | 근처 아군 버프(공격·속도↑) | 오라 버퍼 |
| 저격 개미/스나이퍼(신규) | 초장거리 저격 | 백라인 저격 |
| 여왕 개미/보스(신규) | 대형 광역 | 결전 병기 |
| 메카 개미 | 기존(10합체/직접) · 대포 | 대포 전진 + **자동 쉴드**(15흡수, 피격없이 5s→재충전) |
| 메카 인간폼 | 기존 · 공중 | 공중 전진 + **자동 쉴드**(20흡수, 피격없이 6s→재충전) |
| 인간 | WASD 수동 + 아도겐/무기 | 자동 원거리(아도겐) |

> **메카/인간폼 자동 쉴드(배틀·밸런스 조정됨):** 피격 시 쉴드가 데미지 먼저 흡수. **재충전은 "일정 시간 피격을 안 당해야"** 발동(메카 5s·인간폼 6s) — 즉 **교전 중엔 재생 안 됨**(첫 버스트만 막음, 집중포화·고뎀에 뚫림). 오버레이는 기존 E 홀드 수동(전방향 아님). 흡수/파괴는 소유자 권한·방송으로 일관 처리.

> 신규 소환체는 **캔버스 draw 함수를 새로 제작**(개미 베이스 + 액세서리) — 오버레이·배틀 공용.

### 오버레이 소환 제한 규칙 (신규)
오버레이(장난용)에서 소환체 스팸 방지 + 정리 규칙:
- **수동 조작형(인간·메카개미·게틀링 등 조작 필요):** **1종류만** 동시 존재. 다른 수동형을 소환하면 **기존 것 삭제**.
- **자동 조작형(개미류·솔저·드론 등):** **최대 2종류**까지 동시 존재, **종류 합산 최대 10마리**. 10마리 초과 소환 시 **가장 먼저 소환한 것부터 순번대로 제거(FIFO)**.
- (배틀 모드는 이 제한과 무관 — 마나·덱으로 통제)

## 9e. 전투 구현 = 오버레이 투사체 재사용 (⚠️ 사용자 강조·필수)

배틀 전투는 **기존 오버레이의 투사체·충돌·관통 규칙·쉴드 방어·땅파임을 그대로 재사용**한다. sim의 즉시 데미지는 폐기. 병렬 시스템 신규 제작 금지.

**투사체 매핑 (크기/타입 맞으면 기존 재사용, 없으면 신규 제작):**
| 소환체 | 발사 | 투사체 |
|---|---|---|
| 라이플 솔저 | 3연발 직선 | **게틀링 총알(`gbullets`)** 재사용 — 크기가 맞음 |
| 메카 개미 | 대포 | 기존 `fireMechaShell`(`mshells`) 재사용 |
| 메카 인간폼 | 에너지포 | 기존 `fireEnergyCannon`(`energyShots`) 재사용 |
| 인간 | 아도겐 | 기존 `fireAdogen`(`hbullets`) 재사용 |
| 드론 | 소형 총알 | `gbullets` 재사용(공중) |
| 저격 개미 | 고속 강탄 | 라이플 총알(`hbullets`) 고속 재사용 |
| 얼음 개미 | 둔화탄 | `gbullets` + 명중 시 둔화 효과 |
| **수류탄 솔저** | 포물선 착탄 폭발 | **신규 투사체 제작**(기존 수류탄 디자인 없음) |
| 여왕(보스) | 대형 포탄 광역 | 포탄 확대 or 신규(필요 시) |
| 카미카제 | 자폭 | 기존 폭발(`blast`/`nukeBlast`) |
| 메딕 | 힐(투사체X) | 초록 십자 이펙트 |

**덱 무기(가챠 무기)는 오버레이 무기를 "그대로" 사용 (⚠️ 필수):**
- **발사는 단축키로**(클릭 X — 오버레이 기능 상당수가 커서 클릭 발동이라 부적절). 배틀 중 `fire-slot`(키) → `battleWeaponFire`.
- **키 배정 = 배틀 덱 순서(오버레이 슬롯 무시):** 오버레이 무기 슬롯 설정과 무관하게, **배틀 덱 무기 순서대로 앞 단축키 2개**에 고정 배정 — 키1(예 Alt+Z)=배틀 덱 무기1, 키2(Alt+X)=배틀 덱 무기2. HUD 무기 카드는 그 위치 키를 표시.
- **단축키 라벨 단일 소스:** `slotKeyLabel(i)` → 설정에서 단축키(키 문자)만 바뀌어도 무기설정 팝업·배틀 HUD가 자동 최신화(배틀 중이면 HUD 재빌드).
- 배틀 HUD의 무기 발동 = **실제 오버레이 무기 함수 호출**(미사일 `fireHoming`, 게틀링 `deployGatling`, 쉴드 `activateShield`, 그물 `toggleNetAim`, 블랙홀 `activateBlackhole`, 낙뢰 `fireBolt`). 배틀 전용 간이 발사(폐기됨) 금지.
- **미사일 = 오버레이와 100% 동일:** 캐릭터(내 기지 고양이)에서 발사 → **커서 추적** → **여러 개 발사 시 합체(파워↑)** → **10 합체 = ☢핵** → **상대 핵과 내 핵이 만나면 리틀보이**(멀티). 크기·연출 전부 오버레이 그대로.
- **배틀 데미지 연동:** 오버레이 무기가 폭발/명중하는 지점(`explode`/`nukeBlast`/미사일·게틀링 스텝)에서 `battleHitAt()`로 **적 유닛(side1)·적 기지**에 데미지. 파워는 배틀 유닛 HP 스케일에 맞춰 `×BATTLE_W_MULT(=4)` 증폭.
- **마나만 배틀 코스트:** 발사 시 무기의 마나만 소모(경제 유지), 그 외 메커니즘은 오버레이와 동일.

**유닛 원거리 공격 투사체 규칙:**
- **크기·타입이 맞는 기존 투사체는 그대로 재사용**(솔저=게틀링탄, 메카=기존 포탄 등). **딱 맞는 게 없으면 신규 투사체 디자인 제작**(예: 수류탄 솔저).
- 투사체가 **유닛에 안 맞고 땅에 맞으면 `carveTaskbar`로 땅이 파임** → 지상 유닛 낙하 전략. (기존 투사체가 이미 이렇게 동작)
- **작업표시줄 복원(복구) 버튼 = 마나 코스트 1** (`resetTaskbarDig` 재사용), HUD에 배치.
- **배틀 시작 시 작업표시줄은 "복원된(깨끗한)" 상태로 시작** — 시작 전 파인 상태를 기억/이월하지 않음. (종료 후 원래 오버레이 상태 복귀)
- **메딕 힐 연출:** 초록 십자가가 **메딕 본인 + 회복받는 대상 유닛 양쪽에** 표시.
- 메카 자동 쉴드가 이 "진짜 투사체"를 막는 게 눈에 보여야 함(즉시뎀이면 안 보임).

## 10. 멀티 동기화 (일관성 원칙 준수)

> 원칙: 모든 상호작용은 모든 클라이언트에서 **동일하게** 해결. 비소유자는 남의 것을 로컬로 못 지우고 신호만 보냄. (owner-authoritative)

- 각자 **자기 유닛만 시뮬**하고 상태(type·L·hp) 브로드캐스트(~15Hz).
- 전투 데미지는 **`unit-hit` 릴레이**로 소유자가 확정(기존 hit 패턴 재사용).
- 유닛 사망은 **`boom`처럼 방송**(비소유자 로컬 삭제 금지).
- **새 릴레이 필요 목록(초안):** `battle-req`, `battle-acc`, `battle-decline`, `battle-start`, `battle-end`, `unit-spawn`, `units`(위치 배치), `unit-hit`, `unit-die`, `bet-settle`.
- 서버(`server/server.js`)에 위 메시지 relay 추가 + **재시작** 필수.

### ⚠️ 충돌 일관성 — 최우선 (사용자 반복 강조, 배틀에서 특히 중요)
**모든 소환체·무기의 충돌/피격/사망/기지피해/구멍 낙하는 두 클라이언트에서 반드시 동일하게 해결.** 한쪽에서만 터지거나 죽으면 안 됨.
- 각자 **자기 유닛만 시뮬 + 결과 이벤트 방송**(hit/die/base-hit/boom). 상대는 신호로만 반영, 로컬 추측 삭제 금지.
- 위치·구멍·속도 전부 정규화(`L`) → 해상도 무관 동일 판정.
- **신규 메커니즘(메딕 힐·카미카제 자폭·프리저 둔화·드론 비행·보스 광역 등)도 기본적으로 릴레이 기반 소유자 권한으로 구현** — 로컬 전용 처리 금지. 새 유닛/무기는 예외 없이 이 모델에 편입.
- (→ [[multiplayer-consistency-invariant]] 메모리)

- **B. 배틀 정규화(정석):** 랭크/경쟁전에선 **모든 유닛을 공통 레벨(예: 만렙 또는 기준 레벨)로 고정** → 수치는 캐주얼/PvE·연출용, 경쟁은 순수 실력·조합. (가장 안전)

- **D. 매치 조건:** 신청 시 **레벨 합/평균이 비슷한 상대끼리** 매칭하거나, 베팅에 레벨차 핸디캡.
