## 1. 개요 / 목표

- 냥코대전쟁 스타일 **1v1 라인 대전**. 두 플레이어가 작업표시줄(레인)의 양 끝 진영에서 시작.
- **마나 코스트**로 유닛을 소환 → 유닛은 상대 기지 방향으로 **자동 진군** → 만나면 전투.
- 기존 **무기(미사일·게틀링 등)**로 다가오는 적 유닛을 **방어**.
- **상대 기지(고양이 HP) 파괴 = 승리.**
- 유닛은 **가챠**로 획득, **덱 5장**으로 편성해 입장.
- **베팅:** 신청 시 카운트 재화를 걸고, 이긴 사람이 가져감.

---

## 2. 매치 흐름

1. **신청:** 신청자가 상대 피어 + **베팅액** 지정 → `battle-req` 릴레이.
2. **수락/거절:** 상대에게 팝업 표시. 수락 → `battle-acc`, 거절 → `battle-decline`.
3. **진입:**
   - 두 참가자 외 **모든 피어 숨김 + 상호작용 차단** (배틀 모드 플래그 on).
   - 각 참가자 고양이가 자기 **레인 끝(기지)**으로 자동 이동.
   - 마나 0에서 시작, 베팅액 양쪽 escrow(예치).
4. **전투:** 유닛 소환/무기 사용으로 상대 기지 HP를 깎음.
5. **종료:** 한쪽 기지 HP 0 → **윈/로즈 연출 3초** → 원래 방으로 복귀(피어 복원), 베팅 정산.
6. **정산 규칙 (확정):**
   - **정상 종료:** 양쪽이 보고한 결과가 **일치할 때만** 승자에게 팟 정산.
   - **본인 의사로 이탈**(나가기/포기 버튼, 배틀 모드 명시적 종료) → 남은 유저 **승리 판정**.
   - **시스템 오류·비자발적 팅김**(예상치 못한 ws close/타임아웃, leave 메시지 없이 끊김) → **베팅 무효(환불)**.
   - 판별 휴리스틱: 명시적 `battle-leave` 수신 = 자발적 이탈(패배). 그 외 갑작스런 연결 끊김 = 비자발적(무효). 완벽 판별은 불가하므로 이 기준을 사용.

## 3. 좌표계 & 해상도 독립 (핵심 설계)

플레이어마다 화면 해상도가 달라 픽셀 거리가 다름 → **픽셀을 와이어에 절대 싣지 않는다.**

- **레인 좌표 `L ∈ [0,1]`**: `L=0` = side0 기지, `L=1` = side1 기지. **와이어에는 `L`만 전송.**
- **렌더 시 각자 화면폭에 매핑 + 관점 뒤집기:**
  - 내가 side0 → `screenX = L × laneWidthPx`
  - 내가 side1 → `screenX = (1 − L) × laneWidthPx` (내 기지가 항상 내 화면의 같은 쪽에 보이게)
- **속도/사거리 단위는 픽셀이 아니라 레인비율.** 예: `speed 0.15` = 초당 레인의 15% 이동. → 화면 넓이와 무관하게 **레인 횡단 시간이 모두 동일.**
- **side 결정:** 신청자=side0, 수락자=side1 (또는 netId 낮은 쪽=side0). 리틀보이식 결정적 tie-break.

## 4. 마나 / 코스트

- 마나 **최대 10**, 회복 **+0.5/초** (0→풀차지 20초).
- 유닛 소환 = 해당 유닛 `cost` 만큼 마나 소모.
- 무기 시스템도 마나 코스트 사용(미사일 0.5/발 등).

---

## 5. 덱

- **덱 슬롯 5장.** 보유 유닛 중 5개 골라 입장.
- 배틀 중 덱 5칸을 버튼/숫자키(1~5)로 소환 (마나 충분 시).

## 6. 유닛 데이터 스키마 (하드코딩 금지 → 데이터 정의)

유닛 = **데이터 한 줄**, **범용 시뮬레이터 1개**가 이동·타겟·공격 처리.
속도/사거리/범위는 전부 **레인비율 단위**(3장 참고).

```js
// cat:'unit' = 덱에 넣는 소환체 / cat:'weapon' = 카운트로 사는 무기(덱 X, 누구나)
const UNITS = {
  ant:       { name:'개미',       cat:'unit', rarity:'basic', owned:true, cost:1, hp:20,
               speed:0.18, atk:{ type:'melee', dmg:5,  range:0.02, cd:0.6 } },
  shielder:  { name:'쉴더',       cat:'unit', rarity:'R',     cost:2, hp:40,
               speed:0.10, atk:{ type:'none' }, shield:{ absorb:10 } },        // 10뎀 막고 파괴, 공격력 없음
  rifleman:  { name:'라이플 솔저', cat:'unit', rarity:'R',     cost:2, hp:22,
               speed:0.13, atk:{ type:'proj', dmg:4, range:0.22, cd:1.2, burst:3, projSpeed:1.4 } }, // 3연발
  grenadier: { name:'수류탄 솔저', cat:'unit', rarity:'SR',    cost:3, hp:26,
               speed:0.11, atk:{ type:'aoe', dmg:8, range:0.20, cd:1.8, aoeR:0.06, arc:true } },     // 범위
  mechaAnt:  { name:'메카 개미',   cat:'unit', rarity:'SR',    cost:5, hp:80,
               speed:0.09, atk:{ type:'proj', dmg:12, range:0.30, cd:0.5 }, art:'mecha' },  // 기존 메카 그림 재사용, 개미대포
  mechaHuman:{ name:'메카 인간폼', cat:'unit', rarity:'SSR',   cost:7, hp:120,
               speed:0.12, atk:{ type:'proj', dmg:15, range:0.28, cd:0.7 }, art:'human' },  // 기존 인간폼 그림 재사용
}

// 덱 불필요, 카운트로 구매한 유저만, 마나 코스트로 사용
const WEAPONS = {
  gatling: { name:'게틀링건', cat:'weapon', mana:7,   place:'base-fixed', aim:'cursor' }, // 내 책상 위 고정, 커서 방향 발사
  missile: { name:'미사일',   cat:'weapon', mana:0.5, merge:true },                       // 지금 로직/합체 유지
}
```

**시뮬레이터 개념:** 매 프레임 각 유닛 `L += speed*dt`(기지 방향), 사거리 내 가장 가까운 적 유닛/기지 찾으면 `atk.type`별로 `cd`마다 공격. 렌더는 `art` 필드로 기존 draw 함수에 위임. → **유닛 추가 = 데이터 추가**, 코드 무증가.

> `rarity` 값은 예시(가챠 밸런스 잡을 때 조정).

### 추가 예정 유닛 후보 (역할 다양성 + 레어도 스프레드)
- 🩹 **메딕** — 주변 아군 회복(서포터)
- 🎯 **스나이퍼** — 초장사거리·저연사·고뎀·약체
- 🚁 **드론/비행** — 지상 근접 무시, 원거리/대공만 피격 (카운터 레이어)
- 💣 **카미카제** — 접근 후 자폭 광역
- 🚩 **커맨더/버퍼** — 오라로 근처 아군 공격·속도↑
- 🏗 **워커** — 정지형, 마나 회복/상한↑ (경제 전략)
- ❄️ **프리저** — 범위 내 적 이동 둔화(컨트롤)
- 👑 **SSR 보스급** — 대형 광역 딜러

## 7. 소환체 vs 무기 시스템 구분

- **유닛(cat:'unit'):** 덱에 편성, 가챠로 획득. 자동 진군.
- **무기(cat:'weapon'):** 덱 불필요, **카운트로 구매한 유저면 누구나** 배틀 중 마나 코스트로 사용. (게틀링·미사일 등 기존 무기 체계 재활용)

---

## 8. 가챠 / 컬렉션

- **가챠 재화 = 전용 재화(💎 젬, 임시명)** — **카운트와 분리.** 승리/일일 등으로 지급(획득처는 추후 확정). 전투 재화(카운트)와 다른 통화를 써서 밸런스 조절이 쉬움.
- 유닛은 **가챠로 획득**. 레어도 가중치(basic/R/SR/SSR).
- **중복 = 레벨업**(HP/공격 소폭↑), **피티(천장)** 시스템.
- 보유 유닛/레벨 **localStorage 영속**.
- **레어도 = 파워가 아니라 다양성/상황 특화.** 강함은 마나 코스트에 묶어 PvP 페이-투-윈 방지.

## 9. UI / HUD

- 하단: **마나 바(10칸)** + **덱 5칸 버튼**(코스트 표시, 클릭/숫자키 1~5).
- 레인 양 끝: **양측 기지 HP 바**, 상단: **베팅 팟**.
- 모드 활성 신호: 작업표시줄 라인에 은은한 배틀필드 띠/살짝 어둡게. 다른 피어 숨김.
- 소환은 항상 **내 기지 끝**에서.

---

## 10. 멀티 동기화 (일관성 원칙 준수)

> 원칙: 모든 상호작용은 모든 클라이언트에서 **동일하게** 해결. 비소유자는 남의 것을 로컬로 못 지우고 신호만 보냄. (owner-authoritative)

- 각자 **자기 유닛만 시뮬**하고 상태(type·L·hp) 브로드캐스트(~15Hz).
- 전투 데미지는 **`unit-hit` 릴레이**로 소유자가 확정(기존 hit 패턴 재사용).
- 유닛 사망은 **`boom`처럼 방송**(비소유자 로컬 삭제 금지).
- **새 릴레이 필요 목록(초안):** `battle-req`, `battle-acc`, `battle-decline`, `battle-start`, `battle-end`, `unit-spawn`, `units`(위치 배치), `unit-hit`, `unit-die`, `bet-settle`.
- 서버(`server/server.js`)에 위 메시지 relay 추가 + **재시작** 필수.

## 11. 개발 순서 (phase)

1. **유닛 데이터 스키마 + 로스터 정의** (작음, 기반)
2. **가챠 + 컬렉션 + 덱 편성 UI** (전투 없이 독립 완성 가능) ← 여기부터 착수
3. **배틀 MVP**: 레인(정규화 좌표)·마나·덱 소환·자동 전투(릴레이)·기지 HP·윈/로즈·복귀
4. **베팅** 정산
5. **유닛 확장 + 밸런스** (추가 후보 유닛, 레어도/천장 튜닝)
