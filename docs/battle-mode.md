## 1. 개요 / 목표

- 냥코대전쟁 스타일 **1v1 라인 대전**. 두 플레이어가 작업표시줄(레인)의 양 끝 진영에서 시작.
- **마나 코스트**로 유닛을 소환 → 유닛은 상대 기지 방향으로 **자동 진군** → 만나면 전투.
- 기존 **무기(미사일·게틀링 등)**로 다가오는 적 유닛을 **방어**.
- **상대 기지(고양이 HP) 파괴 = 승리.**
- 유닛은 **가챠**로 획득, **덱 5장**으로 편성해 입장.
- **베팅:** 신청 시 카운트 재화를 걸고, 이긴 사람이 가져감.

## 2. 매치 흐름

1. **신청:** 신청자가 상대 피어 + **베팅액** 지정 → `battle-req` 릴레이.
2. **수락/거절:** 상대에게 팝업 표시. 수락 → `battle-acc`, 거절 → `battle-decline`.
3. **진입:**
   - 두 참가자 외 **모든 피어 숨김 + 상호작용 차단** (배틀 모드 플래그 on).
   - 각 참가자 고양이가 자기 **레인 끝(기지)**으로 자동 이동.
   - 마나 0에서 시작, 베팅액 양쪽 escrow(예치).
4. **전투:** 유닛 소환/무기 사용으로 상대 기지 HP를 깎음.
5. **종료:** 한쪽 기지 HP 0 → **윈/로즈 연출 3초** → 원래 방으로 복귀(피어 복원), 베팅 정산.
6. **정산 규칙 (확정):**
   - **정상 종료:** 양쪽이 보고한 결과가 **일치할 때만** 승자에게 팟 정산.
   - **본인 의사로 이탈**(나가기/포기 버튼, 배틀 모드 명시적 종료) → 남은 유저 **승리 판정**.
   - **시스템 오류·비자발적 팅김**(예상치 못한 ws close/타임아웃, leave 메시지 없이 끊김) → **베팅 무효(환불)**.
   - 판별 휴리스틱: 명시적 `battle-leave` 수신 = 자발적 이탈(패배). 그 외 갑작스런 연결 끊김 = 비자발적(무효). 완벽 판별은 불가하므로 이 기준을 사용.

## 3. 좌표계 & 해상도 독립 (핵심 설계)

플레이어마다 화면 해상도가 달라 픽셀 거리가 다름 → **픽셀을 와이어에 절대 싣지 않는다.**

- **레인 좌표 `L ∈ [0,1]`**: `L=0` = side0 기지, `L=1` = side1 기지. **와이어에는 `L`만 전송.**
- **렌더 시 각자 화면폭에 매핑 + 관점 뒤집기:**
  - 내가 side0 → `screenX = L × laneWidthPx`
  - 내가 side1 → `screenX = (1 − L) × laneWidthPx` (내 기지가 항상 내 화면의 같은 쪽에 보이게)
- **속도/사거리 단위는 픽셀이 아니라 레인비율.** 예: `speed 0.15` = 초당 레인의 15% 이동. → 화면 넓이와 무관하게 **레인 횡단 시간이 모두 동일.**
- **"동일 거리"의 정의:** 맞춰야 하는 건 픽셀 거리가 아니라 **레인 진행도/시간**. 픽셀 거리는 화면마다 달라도 게임적으로는 완전히 같다.
- **side 결정:** 신청자=side0, 수락자=side1 (또는 netId 낮은 쪽=side0). 리틀보이식 결정적 tie-break.

### 지형(땅 파임) & 구멍 — 배틀에서도 유지
- 작업표시줄 파임(carve)·실제 구멍 그대로 유지. **참호로 적 지상 진군을 막는 전략 요소**(내 유닛도 빠지는 양날).
- **구멍 위치·크기도 `L`(정규화)로 표현·방송.** → 양쪽 화면에서 "같은 L의 유닛이 같은 구멍에 빠진다"가 동일하게 참(해상도는 픽셀 수만 바꿀 뿐 판정 불변).
- **지상 유닛:** 구멍 만나면 낙하/제거(기존 `mechaOverHole` 로직 재사용). **공중 유닛(`flying:true`):** 구멍 무시.
- **낙하 판정 = 소유자 권한:** 공유된 L 구멍 데이터로 소유자가 판정 → 결과 방송(일관성 원칙). 세로 낙하 애니메이션은 로컬 연출, "빠졌다/제거"는 모두 동일.

## 4. 마나 / 코스트

- 마나 **최대 10**, 회복 **+0.5/초** (0→풀차지 20초).
- 유닛 소환 = 해당 유닛 `cost` 만큼 마나 소모.
- 무기 시스템도 마나 코스트 사용(미사일 0.5/발 등).

---

## 5. 덱

- **덱 슬롯 5장.** 보유 유닛 중 5개 골라 입장.
- 배틀 중 덱 5칸을 버튼/숫자키(1~5)로 소환 (마나 충분 시).

유닛 = **데이터 한 줄**, **범용 시뮬레이터 1개**가 이동·타겟·공격 처리.
속도/사거리/범위는 전부 **레인비율 단위**(3장 참고).

**실제 데이터 정본 = `renderer/battle/units.js`** (`window.BattleData`). 아래 표는 밸런스 초안(사람이 보는 요약).

**시뮬레이터 개념:** 매 프레임 각 유닛 `L += speed*dt`(기지 방향), 사거리 내 가장 가까운 적 유닛/기지 찾으면 `atk.type`별로 `cd`마다 공격. 렌더는 `art` 필드로 기존 draw 함수에 위임. → **유닛 추가 = 데이터 추가**, 코드 무증가.

## 7. 소환체 vs 무기 시스템 구분

- **유닛(cat:'unit'):** 덱에 편성, **가챠로 획득**. 자동 진군.
- **무기(cat:'weapon'):** 덱 불필요, 배틀 중 마나 코스트로 사용. **획득도 가챠로 전환**(아래 참고).

### 기존 무기 → 가챠 전환 (경제 변경)
**지금 존재하는 오버레이 무기들의 획득을 "카운트 구매"에서 "가챠"로 전환한다.** (카운트 구매 폐지, 대신 카운트→젬 치환)
이는 오버레이 게임 무기에도 적용되는 큰 변경 → dev 게이트 뒤에서 함께 개발, 완성 전 미배포.

## 8. 가챠 / 컬렉션

- **가챠 재화 = 💎 젬** (임시명, 카운트와 분리). **카운트 10,000 → 젬 1개로 치환**(교환 버튼). 1회 뽑기 = 💎1.
- **진입:** 상점에 **[소환] 버튼** 추가 → 누르면 **가챠 팝업**이 뜬다. 뽑기는 **1회씩만**(멀티 뽑기 없음).
- **풀:** 기본지급(개미·미사일) 제외한 **유닛 + 무기 전체**가 한 풀에 섞여 나옴(추후 배너 분리 가능).
- **희귀도 4단계 & 확률(tier):** 일반 45% / 고급 40% / 희귀 13% / 전설 2%. **tier 선택 후 그 안에서는 균등 분배(엔빵)** → 같은 희귀도에 항목이 늘면 개별 확률 = tier% ÷ 개수. (천장은 추후)
- **희귀도별 연출 구분 (내가 제작):**
  - 🩶 **일반** — 흰 섬광 + 카드 슥 등장(짧게)
  - 🔵 **고급** — 파란 빛기둥 + 반짝
  - 🟣 **희귀** — 보라 소용돌이 + 파티클 확산
  - 🌟 **전설** — 금/무지개 폭발 + 화면 흔들림 + 개미 실루엣 극적 등장
- **중복 획득 = 강화 재료(🔩 강화 부품, 임시명) +1 지급** — 레벨업 대신, 이 재료로 유닛/무기를 **업그레이드**(구체 내용 추후 전달). 피티(천장) 추후.
- 보유 유닛·무기 + 강화 재료 보유량 **localStorage 영속**.
- **레어도 = 파워가 아니라 다양성/상황 특화.** 강함은 마나 코스트에 묶어 PvP 페이-투-윈 방지.

> 데이터 정본: `renderer/battle/units.js`의 `RARITY`(가중치/색/연출키), `GEM`(치환·뽑기 비용), `UPGRADE`(중복 시 강화 재료).

## 9. UI / HUD & 모드 표현 방식

**배틀 모드 = 새 창이 아니라 "같은 오버레이의 모드 전환(토글)".** 투명 오버레이 창 그대로, `battleMode` 상태가 켜지며 규칙/UI만 바뀐다. 바탕화면은 계속 보임(살짝 딤).

**토글 ON 시 변화:**
- 다른 피어 전부 숨김(나+상대만).
- 내 고양이 = 레인 끝 기지에 고정(자유 WASD 로밍 OFF).
- HUD 등장: 하단 **마나 바(10) + 덱 5칸(클릭/숫자키 1~5) + 무기 버튼(구매자만)**, 양 끝 **기지 HP 바**, 상단 **베팅 팟**.
- 입력 전환: 클릭/숫자키 = 소환. 소환체는 내 기지에서 나와 자동 전진.
- 딤 약 30~40%(모드 신호 + 가독성). 바탕화면 여전히 보임.

**진입 연출:** 수락 → "배틀 시작" 배너 → 두 고양이 양 끝으로 슬라이드 → 딤 페이드인.
**종료 연출:** 기지 HP 0 → 윈/로즈 3초 → 딤 페이드아웃 → HUD 제거 → 피어 복원 → 자유 오버레이 복귀.

## 9b. 오버레이 ↔ 배틀 모드 관계 (중요 원칙)

**컬렉션은 공유, 동작은 컨텍스트별로 다르다.** 배틀은 "모드"이므로 홍고캣 기본 오버레이를 항상 참고해 설계한다.
- 예) **개미** — 오버레이: 업그레이드 시 소환 개수↑, 10마리 모이면 메카로 합체(현행 유지). 배틀: 덱 유닛으로 마나 1기씩 소환·자동 전진, **메카 개미는 별도 덱 유닛**(자동 합체 없음).
- 예) **미사일** — 오버레이: 업그레이드 시 개수↑ + 합체(현행 유지). 배틀: 무기(마나 0.5/발)로 발사 수/합체 임계에 업그레이드 반영.
- 업그레이드 수치는 **컨텍스트별로 적용**(오버레이=개수/합체, 배틀=유닛 스탯). 하나의 수집품이 두 모드에서 각자 규칙으로 동작.

### 오버레이 전투 AI (추가 소환체용 — 추후 구현)
추가 소환체를 만들면 **오버레이에서도 장난용 무기로** 쓸 수 있어야 함. 동작 규칙:
- **WASD로 조종하는 유닛(인간형 등)**: 지금처럼 수동 조종.
- **공격 기능이 있는 그 외 소환체**: 자동으로 **가장 가까운 상대방 캐릭터**를 공격.
- **근접 전용(개미류)**: 지금처럼 작업표시줄 위를 자유 이동하다 **상대 소환체가 나오면 교전**.
(추가 소환체는 배틀 모드 어느정도 완성 후 만들며, 그때 이 AI를 함께 구현.)

## 9c. 컬렉션 UI

- 전체 소환체 + 무기를 **등급별로 나열**하는 별도 화면.
- **미획득 = 딤 처리(어둡게)**, **획득 = 활성** + 현재 **업그레이드 수치** 표시.
- 진입: 상점/가챠 근처 버튼. (Phase 2 포함)
- 소환은 항상 **내 기지 끝**에서.

## 10. 멀티 동기화 (일관성 원칙 준수)

> 원칙: 모든 상호작용은 모든 클라이언트에서 **동일하게** 해결. 비소유자는 남의 것을 로컬로 못 지우고 신호만 보냄. (owner-authoritative)

- 각자 **자기 유닛만 시뮬**하고 상태(type·L·hp) 브로드캐스트(~15Hz).
- 전투 데미지는 **`unit-hit` 릴레이**로 소유자가 확정(기존 hit 패턴 재사용).
- 유닛 사망은 **`boom`처럼 방송**(비소유자 로컬 삭제 금지).
- **새 릴레이 필요 목록(초안):** `battle-req`, `battle-acc`, `battle-decline`, `battle-start`, `battle-end`, `unit-spawn`, `units`(위치 배치), `unit-hit`, `unit-die`, `bet-settle`.
- 서버(`server/server.js`)에 위 메시지 relay 추가 + **재시작** 필수.

## 11. 개발 순서 (phase)

1. **유닛 데이터 스키마 + 로스터 정의** (작음, 기반)
2. **가챠 팝업 + 컬렉션 UI + 젬 치환 + 덱 편성** (전투 없이 독립 완성 가능) ← 여기부터 착수
   - 상점 [소환] 버튼 → 가챠 팝업(1뽑기·희귀도별 연출) / 중복 시 강화 재료
   - 컬렉션 UI(전체 목록·등급·미획득 딤·업그레이드 수치)
   - 카운트→젬 치환, 1회 초기화 마이그레이션
3. **배틀 MVP**: 레인(정규화 좌표)·마나·덱 소환·자동 전투(릴레이)·기지 HP·윈/로즈·복귀
4. **베팅** 정산
5. **유닛 확장 + 밸런스** (추가 후보 유닛, 레어도/천장 튜닝)

전부 **dev 게이트 뒤**에서. 완성 전까지 미배포.
