# HongGoCat — 전체 기획서 / 프로젝트 개요

> 현재까지(v0.6.5) 구현된 홍고캣의 **정본(source of truth)** 문서.
> 배틀 모드(개발 중)는 [battle-mode.md](battle-mode.md) 참고.
> ※ 코드가 계속 바뀌므로, 세부 수치는 인용 전 실제 코드 확인 권장.

---

## 1. 개요

- **HongGoCat** = 전체화면·투명·클릭 통과 **데스크톱 오버레이 "봉고 캣" 게임**.
- 키보드/마우스 입력에 맞춰 고양이가 두드리는 봉고캣 연출 + **실시간 WebSocket 멀티플레이어 전투**.
- 무기/소환체·상점 경제·업적·**개미 메카 + 인간형(건담) 변신**까지 갖춘 캐주얼 대전 오버레이.
- 플랫폼: Windows (Electron, NSIS 설치본, 자동 업데이트).

---

## 2. 기술 스택 / 아키텍처

- **Electron 35** — 메인(`main.js`) / 렌더러(`renderer/`) / `preload.js`.
- 투명·프레임리스·항상 위·**클릭 통과 오버레이**. 커서가 위젯 "hotzone" 위일 때만 상호작용 활성.
- **uiohook-napi** — 전역 키/마우스 입력 훅(main.js). 키 식별정보는 와이어로 안 나감(익명 펄스만).
- **렌더링:** 2-캔버스 (`#stage` 고양이, `#fx` 무기), `let ctx` 스왑 방식.
- **멀티플레이어:** `ws` 릴레이 서버(`server/server.js`), 정규화 좌표(nx/ny 0~1), 방 코드 기반, 최대 12명.
- **자동 업데이트:** `electron-updater` + GitHub Releases(`hrkim123/HongGoCat`).
- **의존성:** electron, electron-builder(dev), electron-updater, uiohook-napi, ws.

---

## 3. 캐릭터 & 커스터마이징

- 기본 캐릭터 = 고양이(`animal: 'cat'`). 책상 위에서 타이핑에 맞춰 두드림.
- 커스터마이징: **skin(털색), pattern(무늬), hat(모자·현재 전부 잠금), shape(커스텀 몸체)**. localStorage 저장, 멀티 시 roster로 공유.
- **캐릭터 HP = 100**(`CAT_HP`). **10초당 +1 자연회복**. HP 0 → "완전 파괴". 상점에서 HP 충전(🪙500).
- 멀티 다중 모니터: `curDisp`로 표시 디스플레이 지정, `Ctrl+Shift+.`로 순환.

## 4. 재화 & 상점 (경제)

- **재화 = 카운트(🪙 tapCount)** — 입력/전투로 적립.
- **무기 해금(PRICES): 전부 🪙10,000** (shield, gatling, ant, human, blackhole, lightning, net, adogen).
- **소환 비용(USE_COST, 해금 후에도 매번 소모):** 블랙홀 🪙1,000. (human·gatling은 현재 무료)
- **낙뢰 충전 업그레이드:** +1단계당 🪙3,000 (단계별 최대 위력↑).
- **HP 충전:** 상점에서 🪙500.
- **인간 소지 무기(별도 상점 목록):** 🗡️칼 5,000 / 🔫권총 10,000 / 🎯라이플 30,000 / 🚀바주카 50,000. + 🔵아도겐(기 모으기, 차지 기반) 해금 10,000.

## 5. 무기 / 소환체 시스템

정의: `WEAPONS = { none, missile🚀, shield🛡, ant🐜, blackhole🕳, gatling🔫, human🕺, adogen🔵, lightning⚡, net🕸️ }`
슬롯 3개(1~3, 조합키+숫자)로 장착해 사용.

| 무기 | 설명 |
|---|---|
| 🚀 **미사일** | 커서 유도. 10개 합쳐지면 핵미사일. 핵 2개 충돌 → **리틀보이 폭탄**(낙하, 데미지 30, 폭발 3배). |
| 🛡 **쉴드** | 육각(벌집) 돔. 날아오는 탄 방어. |
| 🐜 **개미** | 지상 소환체, 적을 물어 공격. **10마리 모이면 개미 메카로 합체.** |
| 🕳 **블랙홀** | 커서에 10초 고정, 60초 쿨. **캐릭터 제외 모든 물체**를 중앙으로 빨아들여 소멸(→ [불변 원칙](#12-핵심-불변-원칙)). |
| 🔫 **게틀링건** | 책상 위 고정 소환, 커서 방향 연사. |
| 🕺 **인간** | 소지 무기(칼/권총/라이플/바주카) 또는 맨손·아도겐 사용. WASD 이동. |
| 🔵 **아도겐** | 기 모으기(Q 홀드) → 차지량에 따라 크기/HP/데미지↑, 지형 파괴. |
| ⚡ **낙뢰** | 커서→작업표시줄로 내리침, 홀드 차지(1~5단계). |
| 🕸️ **그물** | 발사체는 잡되 **메카/인간폼/게틀링건 본체는 못 잡음**(발사되는 것만). |

- **작업표시줄 = 지형**: 무기로 파일 수 있음(carveTaskbar). 플랫폼(사용자가 그린 선) HP 30, 충돌 처리.

## 6. 개미 메카 & 인간형(건담) 변신

- **개미 10마리 합체 → 개미 메카**(마법진 연출, 커서 위치에서 등장).
- **인간형(건담) 변신:** `Ctrl+\`` 토글(메카 활성 중). 두 폼 디자인 모두 완성.
- **무장:** 개미 대포(쿨 0.5s), 에너지포, **요격 미사일**(화면 전체 유도, 모든 투사체 요격), **판넬/벌집 쉴드**(E 홀드 유지·메카 따라다님·커서 방향).
- 메카/인간폼 **색상 = 내 고양이 털색**을 따라감(메탈 틴트).
- 파인 지형(작업표시줄) 반영해 이동/낙하.

## 8. 멀티플레이어

- **서버:** `server/server.js`, 포트 8787, 방 코드로 입장, 최대 12명. 바탕화면 "HongGoCat 서버" 바로가기 → `start-server.bat` → `npm run server`.
  - **서버는 자동 업데이트 안 됨** — server.js 변경 시 **수동 재시작 필수**.
  - loopback(같은 PC) 접속자 = **호스트**(전 무기 해금).
- **좌표:** 정규화(nx/ny). 스무딩(`mergeRemote`) + **데드레코닝 외삽 240ms**.
- **소유자 권한(owner-authoritative) 모델:** 각 클라이언트가 **자기 투사체만** 시뮬·파괴하고, 결과를 **명시적 이벤트로 방송**(`boom` 등) → 모두 동일 연출. 비소유자는 남의 것을 로컬로 못 지움.
- **주요 릴레이 타입:** pos, pulse, throw, missiles, hit, shield, ants, ant-hit, blackhole, dig, gatling, gbullets, gat-hit, mecha, mecha-transform, mecha-hit, mshells, human, human-hit, hbullets, bolt, net, capture, **col-dmg**(투사체 상호 데미지), **boom**(파괴/폭발 방송), **littleboy**, kill, peace, healall, platforms/platdraw/plathp/plat-hit, chat, roster/update.

## 10. 개발자 기능 & 평화 모드

- **개발자 모드:** 환경변수 `HONGGOCAT_DEV=1`로 실행 시 개발자 전용 UI 노출.
- **🔒 전체 무기 잠금(peace):** 개발자 토글 — 나 포함 **모든 유저 무기 사용 불가** + 소환체/작업표시줄 초기화. 릴레이 `peace`로 전파.
- **모두 체력 회복(healall):** 개발자 토글.
- **👁 상대별 흐리게:** 각 상대 캐릭터 옆 버튼으로 해당 상대 캐릭터+무기 투명화(로컬).
- **🕊️ 평화 모드(설정창, 개인용):** 무적 육각 돔(9999 HP) + 무기 사용 불가. 설정 체크박스 → `safemode`.

## 11. 배포 파이프라인 & 업데이트 노트

- **버전 업 배포:** `git commit` → `npm version patch/minor` → `git push origin main --follow-tags` → 태그 `v*` → **GitHub Actions(`.github/workflows/release.yml`)가 Windows NSIS 설치본 빌드 + Release 게시**.
- 클라이언트는 electron-updater로 **GitHub Releases에서 자동 업데이트**.
- **업데이트 노트 팝업:** 버전 변경 시 `CHANGELOG` 기반으로 새 버전 항목 표시(localStorage `lastVersion` 비교).
- ⚠️ 서버는 별도 — server.js 바꿨으면 수동 재시작.

## 12. 핵심 불변 원칙 (반드시 준수)

1. **멀티 일관성:** 모든 상호작용은 **모든 클라이언트에서 동일하게** 해결. "한 화면에서 터졌는데 다른 화면에선 안 터짐" 금지. 새 무기/소환체는 **기본으로** 공유 충돌 시스템에 편입(소유자 권한 + 데미지 릴레이). 서버는 새 메시지 타입을 반드시 relay + 재시작.
2. **블랙홀 규칙:** **캐릭터(고양이)를 제외한 모든 물체**에 흡입력 작용, 중앙에서 소멸. 새 투사체 추가 시 블랙홀 흡입 루프에도 편입.

## 14. 주요 파일 맵

- `main.js` — Electron 메인, 전역 입력 훅, 단축키, hotzone, 버전 IPC.
- `preload.js` — `appVersion` 등 브리지.
- `renderer/app.js` — **메인 로직(~4,100줄)**: 렌더링, 무기/소환체, 메카, 멀티 동기화, 상점, 업적, 업데이트 노트.
- `renderer/index.html` / `style.css` — UI(상점·업적·평화 버튼·업데이트 모달).
- `renderer/settings.html` / `settings.js` — 설정(평화 모드 등).
- `server/server.js` — 릴레이 서버.
- `docs/battle-mode.md` — 배틀 모드 기획(개발 중).
